CHILD Documentation

Welcome to the official documentation for the CHILD programming language. This guide will teach you everything you need to know to start coding with CHILD.

Introduction

CHILD (Computational High-level Instruction Language for Dummies) is a beginner-friendly programming language designed with simplicity at its core. Write code that reads like English, without the intimidating syntax of traditional programming languages.

Philosophy

  • Readable: Code should read like plain English
  • Simple: No confusing symbols or cryptic syntax
  • Intuitive: Commands do exactly what they say
  • Accessible: Anyone can learn to program

Who Is CHILD For?

  • Complete programming beginners
  • People intimidated by traditional syntax
  • Students learning computational thinking
  • Anyone who wants to automate simple tasks
  • Educators teaching programming concepts

Getting Started

Installation

Follow these steps to install CHILD on your machine:

  1. Download child.py from the GitHub repository
  2. Ensure Python 3.6 or higher is installed on your system
  3. Run your first program using the command below
python child.py myprogram.child

Your First Program

Create a file called hello.child with the following content:

say "Hello, World!"

Run it from your terminal:

python child.py hello.child
âœ“ Success!

You've just written and executed your first CHILD program!

Basic Concepts

File Extension

CHILD programs use the .child or .chld file extension.

Case Sensitivity

All CHILD keywords are case-sensitive and must be written in lowercase:

// âœ“ Correct
say "Hello"

// âœ— Wrong
SAY "Hello"

Whitespace

While indentation is recommended for readability, CHILD doesn't strictly require it. However, clean formatting makes your code easier to understand.

Variables

Variables store values that you can reuse throughout your program.

Syntax

remember VALUE as VARIABLENAME

Examples

remember 10 as age
remember "Alice" as name
remember 3.14 as pi
remember 5 + 3 as sum

Variable Names

Rules for variable names:

  • Must start with a letter or underscore
  • Can contain letters, numbers, and underscores
  • Cannot be a reserved keyword
  • Are case-sensitive (age and Age are different)
âœ“ Good Names
remember 25 as userAge
remember "blue" as favoriteColor
remember 100 as total_score
âœ— Avoid
remember 25 as x           // Too vague
remember "blue" as a       // Not descriptive

Data Types

CHILD automatically determines the type of data you're working with.

Numbers

Integers (whole numbers):

remember 42 as wholeNumber
remember -17 as negativeNumber

Decimals (floating-point numbers):

remember 3.14 as pi
remember -0.5 as half

Strings

Text wrapped in double quotes:

remember "Hello" as greeting
remember "123" as textNumber  // This is text, not a number

Lists

Collections of items:

list [1, 2, 3, 4, 5] as numbers
list ["red", "green", "blue"] as colors
list [] as emptyList

Input and Output

SAY - Output

Print values to the screen:

say "Hello!"
say 42
say myVariable

Multiple lines:

say "Line 1"
say "Line 2"
say "Line 3"

ASK - Input

Get information from the user:

ask "What is your name?" and remember it as name
ask "How old are you?" and remember it as age
Note

CHILD automatically converts numeric input to numbers when appropriate.

Operators

Arithmetic Operators

remember 10 + 5 as sum          // 15
remember 10 - 5 as difference   // 5
remember 10 * 5 as product      // 50
remember 10 / 5 as quotient     // 2

Order of operations (same as mathematics):

remember 2 + 3 * 4 as result    // 14 (not 20)
remember (2 + 3) * 4 as result  // 20

Comparison Operators

  • is - equals
  • is not - not equals
  • is bigger than - greater than
  • is smaller than - less than
  • is greater than or equal to - greater than or equal
if age is 18 then
    say "Exactly 18!"
end

if temperature is bigger than 30 then
    say "It's hot!"
end

Lists

Lists store multiple values in an ordered collection.

Creating Lists

list [1, 2, 3, 4, 5] as numbers
list ["apple", "banana", "cherry"] as fruits
list [] as emptyList

Adding Items

list [1, 2, 3] as numbers
add 4 to numbers
// numbers is now [1, 2, 3, 4]

Getting Items

Lists start counting at 0 (zero-indexed):

list ["red", "green", "blue"] as colors
get item 0 from colors as first    // "red"
get item 1 from colors as second   // "green"
get item 2 from colors as third    // "blue"

List Size

list [10, 20, 30, 40, 50] as numbers
size of numbers as count
say count  // Shows: 5

Looping Through Lists

list ["dog", "cat", "bird"] as animals
for each item in animals as animal
    say animal
end

Control Flow

IF Statements

Execute code only when a condition is true:

if CONDITION then
    // code here
end
Example: Age Check
remember 15 as age
if age is bigger than 12 then
    say "You're a teenager!"
end

IF-OTHERWISE (Else)

Execute different code based on whether a condition is true or false:

if CONDITION then
    // code if true
otherwise
    // code if false
end
Example: Pass/Fail
remember 85 as score
if score is bigger than 60 then
    say "You passed!"
otherwise
    say "Try again!"
end

Nested IF Statements

remember 85 as score

if score is bigger than 89 then
    say "Grade: A"
otherwise
    if score is bigger than 79 then
        say "Grade: B"
    otherwise
        say "Grade: C"
    end
end

Loops

REPEAT - Fixed Iterations

Repeat code a specific number of times:

repeat NUMBER times
    // code here
end
Example: Hello 5 Times
repeat 5 times
    say "Hello!"
end

COUNT - Range Loop

Count through a range of numbers:

count from START to END as VARIABLE
    // code here
end
Example: Count 1 to 5
count from 1 to 5 as number
    say number
end
// Output: 1, 2, 3, 4, 5

FOR EACH - Iterate Through Lists

Loop through each item in a list:

for each item in LISTNAME as VARIABLE
    // code here
end
Example: Print Fruits
list ["apple", "banana", "cherry"] as fruits
for each item in fruits as fruit
    say fruit
end

Functions

Functions let you teach CHILD new reusable commands.

Defining Functions

Simple function (no parameters):

learn how to FUNCTIONNAME
    // code here
end

Function with parameters:

learn how to FUNCTIONNAME with PARAM1 and PARAM2
    // code here
end

Calling Functions

do FUNCTIONNAME
do FUNCTIONNAME with VALUE1 and VALUE2

Examples

Example 1: No Parameters
learn how to sayHello
    say "Hello, World!"
    say "Welcome to CHILD!"
end

do sayHello
Example 2: One Parameter
learn how to greet with name
    say "Hello, "
    say name
    say "!"
end

do greet with "Alice"
do greet with "Bob"
Example 3: Multiple Parameters
learn how to add with a and b
    remember a + b as sum
    say "The sum is: "
    say sum
end

do add with 5 and 7

Comments

Comments are notes for yourself that CHILD ignores when running your program.

Syntax

// This is a comment

Examples

// Calculate the total price
remember price * quantity as total

say "Total: "  // Display the result
say total
Best Practices
  • Explain WHY, not WHAT (code already shows what it does)
  • Keep comments concise and relevant
  • Update comments when code changes
  • Don't over-comment obvious code

Best Practices

1. Use Descriptive Names

Choose variable and function names that clearly describe their purpose.

âœ“ Good
remember 25 as studentAge
remember "Mathematics" as courseName
âœ— Avoid
remember 25 as x
remember "Mathematics" as c

2. Break Down Complex Problems

Use functions to split large problems into smaller, manageable pieces.

learn how to processOrder with orderData
    do validateOrder with orderData
    do calculateTotal with orderData
    do sendConfirmation with orderData
end

3. Keep Functions Small

Each function should do ONE thing and do it well. If a function is getting too long, break it into smaller functions.

4. Test Edge Cases

Always consider unusual inputs:

  • What if the list is empty?
  • What if the number is negative?
  • What if the user enters text instead of a number?

Error Messages

CHILD provides helpful error messages to guide you when something goes wrong.

Common Errors

Syntax Error
Oops! Error on line 5: Don't understand this command: sya "Hello"

Fix: Check spelling of commands (should be say)

Variable Not Found
Oops! Error on line 3: name 'userAge' is not defined

Fix: Make sure you use remember to create the variable first

Missing 'end'
Oops! Error on line 10: Missing 'end' statement

Fix: Add end to close your if, repeat, count, or learn block

Examples

say "=== Temperature Converter ==="
ask "Enter temperature in Celsius:" and remember it as celsius

remember celsius * 9 / 5 + 32 as fahrenheit

say "Temperature in Fahrenheit: "

Example 2: Grade Calculator

    if score is bigger than 89 then
        say "Grade: A"
    otherwise
        if score is bigger than 79 then
            say "Grade: B"
        otherwise
            say "Grade: C"
        end

ask "Enter your score:" and remember it as studentScore

Example 3: Shopping List Manager
list [] as shoppingList

say "=== Shopping List Manager ==="

repeat 3 times
    ask "Add an item:" and remember it as item
    add item to shoppingList
end

say "Your shopping list:"
for each item in shoppingList as item
    say item
end
